单点登录SSO

发表于 2017-02-09 | 分类于 4A系统 | | 阅读次数

什么是单点登录

  • 单点登录(Single Sign On),简称为SSO，是目前比较流行的企业业务系统整合解决方案之一。SSO的定义是在多个系统中，用户只需要登录一次就可以访问所有相互信任的应用系统。

    简单的说

  • 单点登录，适合有多个系统的场景，比如现在有OA系统，门户系统，人力资源管理系统，档案管理系统，生产管理系统等等；在一个公司里面，如果一个用户需要登录这么多个系统，每天都要不停的登录登录太浪费时间(另外还有统一用户，统一权限的问题，暂且不谈)，所以单点登录就是，如果用户在门户系统登录了，那么他打开OA系统等其他系统的时候就不用再登录了，只需要登录一次，这就是所谓的单点登录。

    基本机制

    当用户第一次访问应用系统的时候，因为还没有登录，会被引导到认证系统中进行登录；根据用户提供的登录信息，认证系统进行身份校验，如果校验通过，会返回给用户一个认证的凭据——ticket(票据)；用户再访问别的应用系统的时候，就会将这个ticket带上，作为自己认证的凭据，应用系统接收到请求之后会把票据送到认证系统中进行校验，检查票据的合法性。如果校验通过，用户可以在不用再次登录的情况下访问其他的应用系统了。

    实现SSO需要的主要功能：

  • 所有应用系统共享一个身份认证系统。(统一身份认证)
    • 统一的认证系统是SSO的前提之一。认证系统的主要功能是将用户的登录信息和用户信息库相比较，对用户进行登录认证；认证成功后，认证系统应该生成统一的认证标志(ticket 票据)，回还给用户。认证系统还需要对票据进行校验，判断其有效性。
  • 所有应用系统能够识别和提取票据(ticket)信息。
    • 要实现SSO，就必须让应用系统能够识别已经登录过的用户。应用系统应该能对票据进行识别和提取，通过与认证系统通信，能够自动判断当前用户是否登录过，从而完成单点登录功能。
