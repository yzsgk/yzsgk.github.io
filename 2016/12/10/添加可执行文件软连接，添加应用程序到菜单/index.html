添加可执行文件软连接，添加应用程序到菜单

发表于 2016-12-10 | 分类于 Arch/Gentoo/BSD | | 阅读次数

起因

有时候下载的压缩包并没有安装程序，而是直接在控制台下执行./文件名.后缀,
实在是很不方便

如何在控制台任意目录执行

创建一个软连接到/usr/bin目录

  • [sudo] ln -s /可执行/文件/完整路径 /usr/bin/链接文件名

    刷新path配置

  • 然后执行 source /etc/profile 更新一下
  • 这时候不论在控制台哪个目录，直接输入链接文件名就可以执行程序了

应用程序菜单中没有启动图标

手动创建.desktop文件

  • 有时候并不是所有程序都希望在控制台中运行，而应用程序菜单中又没有
  • 这时候就需要我们手动创建.desktop文件了。
  • 用文本编辑器在/usr/share/applications中新建一个xxx.desktop文件
  • 其中加入类似如下内容

    [Desktop Entry]
    Name=Atom
    Comment=A hackable text editor for the 21st Century
    GenericName=Text Editor
    Exec=electron –app=/usr/lib/atom %F
    Icon=atom
    Type=Application
    StartupNotify=true
    Categories=GNOME;GTK;Utility;TextEditor;Development;
    MimeType=text/plain;

  • 其中Exec=是可执行文件的路径和启动参数，也是最重要的一项
  • 保存后就能在程序菜单中找到它了，如果觉得太丑就再研究研究其他项的功能吧！
  • 比如图标，提示信息之类的！
    在i3wm中用dmenu也可以看到添加的应用程序了，
    dmenu_run也可以找到前面添加到/usr/bin/的软连接
    现在我们可以告别在控制台输入一长串的路径，
    而直接在程序菜单或控制台任意目录执行运行命令了！
